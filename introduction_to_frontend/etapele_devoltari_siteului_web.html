Etapele de dezvoltare a site-urilor web

Home

De-a lungul timpului, s-au format mai multe etape principale în dezvoltarea site-urilor web. Acestea sunt următoarele:

  1. colectarea informațiilor;
  2. planificare;
  3. design/proiectare;
  4. scrierea conținutului textului;
  5. codarea și programarea;
  6. testarea, verificarea și postarea;
  7. întreținere, monitorizare și actualizare.

1.Colectarea informațiilor

Etapa inițială în dezvoltarea fiecărui site web este cea care implică colectarea de informații, respectiv cereri. Această fază constă din cercetare și luarea în considerare pentru a crea o imagine cât mai clară a caracteristicilor de bază ale unui site web care trebuie realizat. De aceea, obiectivele de bază ale acestei etape inițiale de dezvoltare a site-urilor web sunt următoarele:

  • determinarea scopului site-ului (ce?);
  • stabilirea obiectivelor de bază (de ce?);
  • selectarea grupului țintă (pentru cine?).

Dacă un site web este dezvoltat pentru un client, majoritatea informațiilor de bază vor veni de la acesta. În orice caz, colectarea informațiilor se face mai ales prin conversație, reflecție și cercetare.

Etapa de colectare a informațiilor este foarte importantă, deoarece toate celelalte etape ulterioare se bazează pe ea. De aceea, lipsa realizării acestei etape introductive de dezvoltare poate crea numeroase probleme majore în următoarele etape de dezvoltare. Acest lucru poate afecta în cele din urmă prețul general al produsului, timpul total de construcție, termenele limită, calitatea...

2. Planificarea

După colectarea informațiilor de bază despre site-ul care trebuie făcut, urmează etapa de planificare. Etapa de planificare depinde direct de etapa de colectare a informațiilor, având în vedere că planificarea proiectului se bazează pe informațiile ce sunt colectate în etapa inițială de dezvoltare.

Etapa de planificare constă în general din trei paşi:

1.Crearea unei structuri de bază a site-ului webpresupune determinarea setului de pagini din care va fi compus site-ul (de exemplu: Acasă, Despre noi, Produsele noastre, Contact etc.). Crearea unei structuri complete a site-ului web, care presupune paginile lui, este foarte des reprezentată folosind așa-numita hartă a site-ului (engl. sitemap).

2.Crearea primelor schițe ale unui mediu de utilizatoreprezintă una dintre acele sarcini care poate fi realizată în etapa de planificare. Spre deosebire de harta site-ului, acest pas oferă informații despre componenta vizuală a site-ului web. În etapa de planificare se recurge la crearea unor modele destul de reduse, în linii mari, care nu se ocupă de detalii vizuale specifice. Acest tip de proiectare se mai numeşte şi wireframe

Wireframe este o reprezentare brută a mediului de utilizator al unui produs software, cu care se prezintă elementele de bază din care va fi alcătuită structura vizuală.

3.Alegerea setului de tehnologii care vor fi utilizate pentru dezvoltareste o altă decizie care este bine a fi luată în etapa de planificare. Dat fiind faptul că în acest moment sunt cunoscute toate cerințele privind diversele funcționalități pe care le va avea site-ul web, este necesar să se decidă tehnologiile care vor fi utilizate pentru dezvoltare.

Dacă se dezvoltă un site web sau o aplicație web mai complexă, alegerea setului de tehnologii care va fi utilizat este mult mai complexă. Astfel de site-uri web au și partea lor back-end.

3. Design/proiectare

Prima etapă în care site-ul web creat prinde contur este faza de proiectare. Este vorba de fapt despre etapa care implică crearea de conținut vizual complet: imagini, diagrame, ilustrații, videoclipuri, logouri...

Pe lângă crearea componentelor vizuale individuale, faza de proiectare implică şi crearea unei reprezentări mult mai detaliate a mediului de utilizator. Ca o platformă inițială este luată o schiță brută, respectiv wireframe-ul realizat în timpul etapei de planificare. Deoarece wireframe-ul caracterizează lipsă de detalii, designerul web creează în această etapă o reprezentare mult mai detaliată a site-ului web care va fi creat. O astfel de reprezentare mai detaliată a mediului de utilizator al unui produs software se numește mockup

4. Crearea conţinutului

Fiecare site web are un anumit conținut text care trebuie pregătit cu atenție. Prin urmare, una dintre etapele dezvoltării site-urilor web este şi cea care se referă la crearea unui conținut text. Este vorba de etapa care nu are un loc strict în cronologia dezvoltării. Acest lucru înseamnă că scrierea conținutului textului se suprapune în general cu etapele de proiectare și dezvoltare. Desigur, este mult mai bine când este pregătit conținutul text înainte de a începe să se lucreze la codare și programare. De aceea, cel mai bine este să se pregătească conținutul text în faza de proiectare, când acesta poate deveni o parte integrantă a unui model sau a unui prototip.

În cazul în care conținutul text al unui site web nu este pregătit în timpul etapelor de proiectare sau dezvoltare, se întâmplă foarte des ca în locul conținutului text propriu-zis să fie folosit așa-numitul text dummy

5.Codare şi programare

Prima etapă în care programatorii front-end și programatorii web îşi încep munca este etapa de codare și programare. Este vorba de etapa în care tot ceea ce este prevăzut în specificaţii este transformat într-un produs concret. Pe baza diferitelor modele menționate anterior, este creat un mediu de utilizare a site-ului web. Aici se adăugă diferite elemente grafice create de designeri, precum și conținutul text.

Dacă este vorba de un site web, respectiv despre o aplicație web mai complexă, dezvoltarea se poate realiza în paralel în două straturi diferite ale aplicației: client și server. Programatorii front-end creează o parte de prezentare folosind HTML, CSS și JavaScript, în timp ce programatorii web se ocupă de logica back-end, care poate implica utilizarea diferitelor limbaje de programare și sisteme de gestionare a depozitului de date.

6. Testare şi publicare

Testarea funcționalității unui site web este strâns legată de etapele de codare și programare. Şi anume, deși testarea este menţionată după etapa de codificare și programare, realitatea este că operaţiunile menționate în practică, în general se suprapun. Cele mai bune practici dictează că testarea trebuie efectuată imediat ce se creează o nouă funcționalitate. Astfel, sarcinile de codare, programare și testare sunt intercalate

Testarea este importantă în special atunci când se dezvoltă site-uri web. Având în vedere că site-urile web există într-un mediu foarte specific și că pentru revizuirea lor se poate utiliza un număr mare de browsere web cu proprietăți diferite, este necesară o testare adecvată. De aceea, este mai bine să testați fiecare secțiune a site-ului web imediat după crearea lui. Pe lângă aceste teste locale minore, testarea este necesară și după finalizarea site-ului web complet.

Publicarea presupune plasarea site-ului web pe un server de pe care va fi disponibil public. Şi anume, după terminarea dezvoltării este necesar ca fișierele site-ului web să fie transferate de pe calculatorul local pe calculatorul destinat special pentru găzduirea site-urilor web. Astfel, se poate concluziona că procesul de publicare presupune copierea fișierelor site-ului web de pe un calculator local pe server.

7. Mentenanță

Crearea și publicarea unui site web reprezintă doar o parte a dezvoltării web. Munca programatorilor front-end și a programatorilor web presupune că un site web odată creat trebuie să funcționeze perfect pe întreaga sa durată de viață. Prin urmare, este imperativă monitorizarea și actualizarea constantă.Ca mediu extrem de dinamic, în care se schimbă rapid tehnologiile și tendințele, web-ul necesită îmbunătățirea constantă vizuală și funcțională a site-urilor. Pe lângă acest lucru, clienții, respectiv abonații primesc foarte des idei și cereri noi după publicarea site-ului, care trebuie implementate prin actualizări periodice.
