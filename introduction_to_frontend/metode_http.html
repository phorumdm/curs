Cu ocazia afișării primei cereri HTTP din rândurile anterioare, ne-am întâlnit pentru prima dată cu termenul de metodă HTTP, deoarece cererea afișată începe prin specificarea unei astfel de metode (GET). Este vorba de una dintre multele metode HTTP care pot fi utilizate la formarea unei cereri HTTP. Astfel, metodele HTTP, pe lângă structura în sine a cererilor HTTP, sunt încă un termen foarte important legat de formarea cererilor HTTP.

Câteva dintre cele mai semnificative metode HTTP sunt:

  • GET – necesită datele unei anumite resurse;
  • HEAD– necesită doar antet HTTP, fără corpul răspunsului;
  • POST – transmite o resursă nouă serverului;
  • PUT – transmite serverului o resursă cu care trebuie să înlocuiască resursa deja existentă;
  • DELETE – șterge resursa de pe server;
  • OPTIONS – necesită de la server să livreze un set de metode HTTP pe care le suportă;
  • TRACE – se utilizează pentru testare și debug (procesul de eliminare a erorilor din cod), serverul returnând clientului cererea pe care a primit-o;
  • PATCH– efectuează modificarea parțială a resursei existente.
