Având în vedere că mediul în care există site-urile web este aproape întotdeauna format dintr-un minim de două calculatoare, acest lucru înseamnă practic că există și două locuri diferite pe care se poate executa o anumită logică de programare. Astfel se poate spune că aplicațiile web sunt formate din două părți de bază:

Câteva dintre cele mai semnificative metode HTTP sunt:

  • partea client;
  • partea server.

Partea de client a aplicațiilor web se execută pe calculatoarele care accesează o aplicație web, mai ales în cadrul browserelor web (engl. web browsers). Acest lucru înseamnă că browserul web este de fapt client. Probabil vă sunt bine cunoscute câteva dintre cele mai populare programe de acest tip - Chrome, Firefox, Safari, Internet Explorer, Opera...

Partea de server a aplicațiilor web există și se execută în cadrul calculatoarelor care sunt altfel numite servere (adesea sunt folosiți termeni precum server HTTP sau server Web). Este vorba de calculatoarele care, prin structura lor de bază, nu sunt foarte diferite de calculatoarele client. Mai mult, fiecare calculator, chiar și cel pe care îl utilizați acum pentru a citi această lecție, poate avea rol de server. Totuşi, în condiții reale, calculatoarele server sunt în general mult mai puternice decât cele utilizate în scopuri personale sau de afaceri, în primul rând pentru a putea servi un număr mai mare de clienți în același timp. Aceasta este încă o caracteristică semnificativă a mediului web pe care trebuie să o înțelegeți. Şi anume, un site web sau o aplicație se află în general pe un calculator server, în timp ce numărul de clienți care accesează un astfel de server pentru a vizita un site sau o aplicație poate fi mult mai mare
